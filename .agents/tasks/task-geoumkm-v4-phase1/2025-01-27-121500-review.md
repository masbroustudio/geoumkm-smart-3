# Next.js 14 frontend scaffold for GeoUMKM v4.0 (v2)

This change replaces the monolithic `index.html` landing page with a full Next.js 14 App Router project, restructures the repository so ML content lives under `ml/`, adds Azure deployment documentation, and rewires the GitHub Actions workflow for static export. All four blocking issues from the prior review have been addressed: `frontend/lib/utils.ts` is tracked, `next.config.js` uses `output: 'export'`, the duplicate Google Fonts `@import` is removed, and auth forms now display a "Coming Soon" banner on submission.

Watch for: **Hardcoded dark mode** remains (accepted for Phase 1 scope). No new blocking issues introduced by the fixes.

**Verdict**: APPROVED

## High-level view

The deployment pipeline is now internally consistent. The workflow builds with `pnpm build`, which under `output: 'export'` produces a static `out/` directory. The Azure Static Web Apps deploy step references `output_location: "out"` with `skip_app_build: true`, uploading pre-built static assets. No server-side features (`next/image` optimizer, API routes, dynamic server rendering) are present, so static export is fully compatible with the current codebase.

The `.gitignore` uses `!frontend/lib/` to carve out the frontend utility directory from the blanket Python `lib/` ignore rule. Confirmed tracked via `git ls-files`.

The auth forms use local `useState` to toggle a "Fitur autentikasi akan segera tersedia" banner on submission, matching the "Coming Soon" pattern used by dashboard placeholders.

The single remaining known gap—hardcoded `className="dark"` on `<html>` with no toggle and hardcoded dark-palette Tailwind classes throughout—was accepted as out of scope for Phase 1.
