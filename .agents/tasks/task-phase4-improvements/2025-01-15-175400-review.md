# i18n, PWA support, and auth scaffolding for GeoUMKM dashboard (v2)

Phase 4 adds a Bahasa Indonesia/English language toggle (LanguageProvider context), Progressive Web App scaffolding (manifest, service worker, offline fallback page), and a mock authentication provider with AuthGuard component. The v1 review identified five issues: stale-HTML caching via cache-first navigation, build-time-only analytics docs, a non-differentiating i18n test, untranslated Indonesian sidebar labels, and a hardcoded `<html lang>` attribute. All five have been addressed in commit `62c5f7f`.

**Watch for:** The service worker cache name (`geoumkm-v1`) is still static — there's no automated deploy-time bumping mechanism, but the network-first navigation strategy makes this a cosmetic concern rather than a correctness one (possible). The `LanguageProvider` renders children without context on the first render before `mounted` becomes true, so any component calling `useLanguage()` during SSR/initial paint receives the default context value (`locale: 'id'`); this is consistent with the ThemeProvider pattern and acceptable for a static export (confirmed).

**Verdict**: APPROVED

## High-level view

The service worker now uses network-first for navigation requests with offline fallback, and cache-first only for static assets. This eliminates the stale-HTML-after-deploy problem from v1.

The analytics documentation now explicitly states that `NEXT_PUBLIC_APPINSIGHTS_KEY` must be present at build time and describes how to wire it through CI/CD.

The i18n test asserts `sidebar.overview` as "Ringkasan" (Indonesian) vs "Overview" (English), proving the toggle produces different output. The Indonesian translation file has been fully localized for sidebar, dashboard, map controls, and export buttons.

The `<html lang>` attribute is updated via an inline script (pre-hydration, reading localStorage) and reactively via `LanguageProvider`'s `useEffect`. This mirrors the theme-flash-prevention pattern already in the codebase.
