# Full kecamatan data expansion — fixes for duplicate keys and filter coverage (v2)

The v1 review raised four issues. Issues 1 (duplicate React keys / incorrect comparison lookups) and 2 (kabupaten filter returning empty for 24 of 27 options) were addressed in commit `cb991a4`. Issues 3 (bundle size) and 4 (cluster name drift) were accepted as known trade-offs. This pass verifies the two fixes and checks for regressions.

**Watch for:** No blocking concerns remain. The composite key fix is correct and complete. The `recommendData` expansion covers all 26 kabupaten with 104 entries.

**Verdict**: APPROVED

## High-level view

The comparison dropdown fix replaces bare kecamatan names with composite `kecamatan|kabupaten` values in both state and option elements, and splits on `|` at lookup time to match against both fields. This eliminates the ambiguity for the 35 duplicate kecamatan names. React keys use the `${k.kecamatan}-${k.kabupaten}` format, which is guaranteed unique across the 595-entry dataset.

The `recommendData` expansion grows from 11 entries covering 3 kabupaten to 104 entries covering all 26 kabupaten (4 entries each). Every kabupaten selection now returns results. The combined filter (kabupaten + jenis_usaha) can still yield zero results for certain combinations since each kabupaten only has 4 entries across a subset of business types, but the empty-state message covers this.

## Composite key and lookup mechanics

State initialization stores the full composite value:

```typescript
const [compareA, setCompareA] = useState(`${kecamatanDetailData[0].kecamatan}|${kecamatanDetailData[0].kabupaten}`);
```

Option elements use `value={`${k.kecamatan}|${k.kabupaten}`}` and `key={`${k.kecamatan}-${k.kabupaten}`}`. The key uses a hyphen separator while the value uses a pipe — the pipe is a safer split delimiter since no kecamatan or kabupaten name contains one (verified).

The lookup destructures and matches on both fields:

```typescript
const [nameA, kabA] = compareA.split('|');
const dataA = kecamatanDetailData.find((k) => k.kecamatan === nameA && k.kabupaten === kabA);
```

Labels passed to the radar chart and comparison table now use `"Kecamatan (Kabupaten)"` format, disambiguating entries like "Cibeureum" that appear in three different kabupaten.

## recommendData coverage alignment

The expanded `recommendData` (104 entries) covers exactly the same 26 kabupaten listed in `kabupatenOptions` (excluding "Semua"). Each kabupaten has 4 entries distributed across different jenis_usaha values, sorted by `avg_score` descending (97.72 to 33.48).
